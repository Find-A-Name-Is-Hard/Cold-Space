# 冰冷宙域 / Cold Space
## 简介
<video src="ReadMe_Content/TrailerForColdSpace2.mp4" controls width="100%"></video>

本游戏尝试将弹幕射击玩法与医学诊断主题主题结合，玩家扮演一名驾驶飞船的医生，需要尝试检测敌人弱点，推理出正确的弱点组合再发动最终一击。

## 操控方式
- 方向键↑↓←→：移动飞船

- 按住Shift：降低移动速度

- Z：射击，命中敌人积攒能量，但不造成伤害

- X+数字1~4：发射一种弱点测试攻击，命中后告知敌人是否有这种弱点

- C+数字1~6：针对不同弱点组合，选择不同最终攻击，如果敌人确实带有这种弱点组合，将成功杀死敌人，否则将会激怒敌人

## 技术亮点：
- **MVC框架**：玩家控制器使用MVC框架分离，增加可维护性
- **对象池**： 
    - `Dictionary<string, Stack<GameObject>>` 以`GameObject`的名字为索引，存储对应的对象池
    - 支持同步一次性加载/协程分批加载，避免卡顿
    - 支持场景加载后预加载对象池内对象，默认使用协程加载
- **Singleton 单例模式**：关卡，输入管理器使用单例模式
- **弹幕发射器**：
    - 支持多类型弹幕生成，旋转发射，追踪玩家（自机狙），弹幕朝向/间隔增加随机值（随机弹）
    - 支持多个弹幕发射器同时发射，组成复杂弹幕
    - 支持使用Scriptable Object编辑弹幕攻击，包括：
        - 单次弹幕攻击使用的不同弹幕发射器
        - 每个弹幕发射器的发射速度，每波次数量，每波次中每个弹幕角度角度间隔，每波次时间间隔，等等

# 困难与反思
- **需要事件总线**：本项目使用`Action<T>`让其他脚本订阅事件，但导致各个脚本之间都需要持有对方的引用，依赖复杂，后期不好维护和扩展。如果引入事件总线作为中介会更加方便
- **对象池**：
    - 对象池本身使用List或Hashset比Stack更方便，如需要销毁子弹时，无法简单地从Stack对象池中查找并将子弹从池中移除，而List或Hashset则支持。同时，这个问题会导致Stack池中可能出现大量已经被销毁的对象，从池中取出时需要额外验证对象是否为空
    - 需要支持协程/异步加载，否则一帧内同时加载几百几千个对象可能会卡顿
    - 对象池使用预制体的ID作为索引比用名字更好，因为名字会被改变，可能会导致找不到池子
